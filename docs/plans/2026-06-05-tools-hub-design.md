# Tools Hub 需求设计

## 背景

参考站点包含网盘搜索、资源论坛、软件文章站和 ReMan 类管理系统。它们常见能力包括分类目录、搜索、投稿、用户入口、签到、热词、排行榜、版权处理和后台管理。为了降低平台封禁、版权和安全风险，本项目不实现网盘爬取、破解软件分发或资源聚合搜索，而是实现一个合规的软件工具导航和公众号文章承接页。

## MVP

- 首页提供工具搜索、分类筛选、平台筛选和精选工具卡片。
- 工具卡片展示官方来源、授权类型、使用场景、风险提示和跳转按钮。
- 公众号承接区展示文章合集，把用户从公众号文章引导到站内专题。
- 投稿表单收集工具官网、授权说明、来源类型和审核备注。
- 合规区提供版权处理、外链审核、不托管文件和数据删除说明。
- 运营区记录论坛、公众号、社群和 SEO 页的投放策略，避免直接采集或搬运第三方内容。

## 信息架构

数据层使用 `src/data/catalog.ts`，包含工具、分类、文章活动和投放渠道。界面层使用单页 React 应用：顶部导航、首屏搜索、筛选栏、工具列表、文章承接、投稿审核和合规说明。

## 后续扩展

- 接入 Supabase/Postgres 保存工具、投稿和审核状态。
- 增加后台登录、角色权限、批量导入和外链健康检查。
- 给每个公众号文章生成独立 slug 页面，支持 SEO 标题和结构化数据。
- 增加版权投诉工单和自动下架流程。
